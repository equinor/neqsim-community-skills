# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please disclose it
responsibly:

1. **Do not** open a public GitHub issue for the vulnerability.
2. Follow Equinor's Responsible Disclosure Policy at
   [https://www.equinor.com/about-us/csirt](https://www.equinor.com/about-us/csirt)
   to report the issue.

The following information helps us triage your report more quickly:

- Type of issue (e.g. code execution, credential exposure, path traversal)
- Full path of the source file(s) related to the issue
- The location of the affected code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce
- Step-by-step instructions to reproduce
- Proof-of-concept or exploit code (if possible)
- Impact of the issue and how an attacker might exploit it

We prefer all communications to be in English.

## Scope and Data Handling

This is a public catalog of reusable NeqSim skills. Never include credentials,
tokens, cookies, internal URLs, host names, plant data, production data, or
document identifiers in issues, pull requests, examples, or test data. Use only
small synthetic examples and public references.
